Project Overview & Data Persistence

The Odometer project was designed to solve a fundamental challenge in application development: **data persistence**. Unlike a simple calculator that resets every time you close it, this odometer uses **Local Storage** (a component in MIT App Inventor) to remember its current count.

This means if the user closes the app, reboots their device, or moves to a different screen and comes back, the odometer will always display the number it was at last. This is a crucial concept for any application that needs to save user progress or settings.

Technical Skills Demonstrated

The development highlighted key skills in managing application data:

  • **Data Persistence:** Implementing 'Load' and 'Save' functions using the local storage component. The 'Save' function runs every time the counter changes.
  • **Counting Logic:** Handling the logic for an odometer display, where incrementing the "ones" column rolls over to increment the "tens" column, and so on.
  • **User Input Handling:** Efficiently connecting button clicks to corresponding numerical operations (counting and reset).
  • **Initialization Logic:** Writing code that runs when the app starts up, checking if a saved value exists and loading it, or setting the counter to zero if it's the first time running the app.

**Counting Logic Blocks:** This complex set of blocks manages the core odometer function, handling the "rollover" from 9 to 0 and incrementing the next digit (e.g., from 09 to 10), ensuring the count is always mathematically correct.
